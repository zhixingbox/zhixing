## 项目简介
**《小红帽与灰太狼：森林守护者》** 是基于经典童话 IP 重构的 AI 驱动型动漫项目，也是一款创新的互动式编程教育产品，通过 xgopilot 工具与 xbuilder 开发平台实现。讲述了一段关于救赎、联盟与成长的奇幻冒险故事。

### 核心故事线
（1）内容

本项目以"森林守护者"为主题，重新演绎了经典的小红帽故事：
- **小红帽**作为见习森林守护者，肩负着守护森林的使命
- **大灰狼**因被黑暗巫师控制而陷入危机
- 通过**勇气、智慧和合作**，主人公们最终建立联盟，共同对抗黑暗势力

（2）（AI 赋能 + 商业亮点）​

- AI 智能技术深度落地​

    智能内容生成：借助 AI 绘画工具批量生成角色动态帧、场景背景与魔法特效，降低美术制作成本​
- 商业价值多维挖掘：

    IP 衍生潜力：基于 “森林守护者” 世界观，可延伸开发 AI 生成式周边（如用户上传照片生成专属守护者形象、AI 定制剧情漫画），形成 “内容 + 工具” 的变现闭环。​

    场景化商业植入：通过 AI 算法分析用户偏好，在非核心剧情节点自然植入品牌合作内容（如魔法草药道具可关联健康食品品牌、森林场景可植入户外用品广告），兼顾用户体验与商业收益。​

    B 端授权变现：项目 AI 交互框架可模块化输出，授权给儿童教育平台、动漫 IP 孵化公司，提供 “AI + 童话” 的标准化解决方案，拓展 To B 商业渠道。​

## 运行说明
1. 环境要求（适配商业部署场景）​

    基础环境：支持 xbuilder 的桌面端（Windows10+/MacOS12+）、浏览器（Chrome90+/Edge90+）、移动端（iOS14+/Android10+），兼容小程序内嵌场景。​
AI 运行依赖：建议服务器端配置 GPU（NVIDIA Tesla T4 及以上），支持 AI 实时交互与内容生成；客户端最低配置：处理器 1.6GHz、内存 2GB，满足轻量化 AI 特效渲染。​
资源要求：确保项目包含 AI 生成资源包（角色动画帧、场景素材、特效模板）、智能交互算法库、音频资源库（背景音乐 + AI 合成音效）。​

2. ​运行说明

    资源要求：确保项目包含所有必要的精灵资源（角色、场景、特效）和音频资源（背景音乐、音效）
    最低配置：处理器 1.6GHz 以上，内存 2GB 以上，网络环境稳定

3. 启动步骤

    （1）项目导入
    
    - 打开 xbuilder 开发平台，选择 "导入项目"，定位到《小红帽与灰太狼：森林守护者》项目文件夹，等待资源加载完成（首次加载可能需要 1-2 分钟）

    （2）资源检查

    - 进入 "资源管理" 界面，确认以下资源已正确导入：
    角色精灵：redHood（小红帽）、wolf（灰太狼）、darkWizard（黑暗巫师）等；
    场景背景：cabin（小屋）、forbidden_forest（禁忌森林）等；
    音频资源：forest_music（森林音乐）、battle_sound（战斗音效）等。

    （3）启动项目

    - 点击平台工具栏中的 "运行" 按钮（或按 F5 键）；首次运行会自动执行onStart初始化函数，加载初始场景 "森林边缘的小屋"；等待初始化完成后即可进入剧情体验。

## 技术特性

### 核心技术栈
- **编程语言**: GoPlus (SPX) - 专为创意编程设计的现代语言
- **开发平台**: GoPlus Builder - 可视化编程环境
- **资源管理**: 基于 JSON 的资源索引系统
- **版本控制**: Git - 支持团队协作开发

### 技术亮点

```go
// 示例：精灵动画与事件驱动编程
onBackdrop "moonlight_lake", => {
    setGraphicEffect ColorEffect, 30
    play "bgm", true
    
    // 角色互动逻辑
    wolf.say "快...快走...我控制不了自己...", 2
    redHood.say "也许...我可以帮你。", 2
    
    // 动画效果
    magicWater.glide 120, 10, 1.5
}
```

**实现功能**：
- ✅ 多场景切换与过渡效果
- ✅ 角色对话系统
- ✅ 精灵移动与动画控制
- ✅ 音效与背景音乐同步
- ✅ 图形特效（颜色、透明度、闪烁等）
- ✅ 事件广播与监听机制
- ✅ 复杂的剧情逻辑编排

```
zhixing/
├── main.spx                 # 主场景控制逻辑

├── wolf.spx / wolf1.spx     # 大灰狼角色脚本
```

## 教育价值

## 快速开始

### 环境要求
- GoPlus 开发环境
- GoPlus Builder（推荐）

### 运行项目
1. 克隆仓库
```bash
git clone https://github.com/zhixingbox/zhixing.git
cd zhixing
```

2. 使用 GoPlus Builder 打开项目

3. 运行 `main.spx` 开始体验完整故事

### 自定义开发
- 修改 `.spx` 文件可调整角色行为和对话
- 替换 `assets/` 中的资源可更换视觉风格
- 添加新的 `.spx` 文件可扩展新角色
